### 1. 오늘 배운 내용 
 - 절차지향 언어와 객체지향 언어
 - 클래스와 객체. 클래스는 설계도, 객체는 실체!
 - 필드와 메서드. 필드는 객체가 무엇을 가지고 있는가, 메서드는 기능.
 - 생성자와 this. 생성자는 초기상태 세팅, this는 자기 자신.
  변수 이름이 겹칠때 this.name은 클래스 안의 변수
 - 추상화. 복잡한 세부사항은 숨기는 것. 내부 구현은 숨김
 - 추상 클래스와 추상 메서드. 추상 클래스는 미완성 설계도, 추상 메서드는 내용 없는 함수.
 - 추상 클래스 선언 후엔 자식 클래스로만 생성해야 함.
 - 상속. 부모의 필드와 메서드를 자식이 물려받아 재사용/확장
 - 캡슐화. 데이터를 외부에서 직접 건드리지 못하게 숨김.
 - 다형성. 하나의 타입으로 여러 객체를 다룰 수 있는 성질.
### 2. 핵심 정리
- 클래스 -> 설계도
- 객체 -> 실제 만들어진 것
- 필드 -> 데이터
- 메서드 -> 기능
- 추상화 -> 복잡한 내부 구현은 숨김처리
- 추상 클래스 -> 미완성 클래스. 선언 후엔 자식 클래스로만 생성
- 추상 메서드 -> 제목만 있는 메서드
- 상속 -> 부모의 기능을 자식이 물려받음
- 캡슐화 -> 누가 함부로 손 못대게 데이터 보호
- 다형성 -> 하나의 타입으로 여러 객체를 다룰 수 있는 성질.
